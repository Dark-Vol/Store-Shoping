# Как запустить проект

## Предварительные требования

- Node.js (v18 или выше)
- npm или yarn
- MySQL база данных

## Шаг 1: Установка зависимостей

### Установка зависимостей для клиента:
```bash
cd client
npm install
```

### Установка зависимостей для сервера:
```bash
cd server
npm install
```

## Шаг 2: Настройка базы данных

1. Создайте базу данных MySQL
2. Настройте подключение в `server/config/db.ts` (или через переменные окружения)

## Шаг 3: Запуск сервера (Backend)

### Вариант А: Запуск в режиме разработки (с автоматической перезагрузкой)
```bash
cd server
npm run dev
```

### Вариант Б: Сборка и запуск в production режиме
```bash
cd server
npm run build    # Компиляция TypeScript в JavaScript
npm start        # Запуск скомпилированного сервера
```

Сервер будет доступен на: **http://localhost:4000**

## Шаг 4: Запуск клиента (Frontend)

В новом терминале:
```bash
cd client
npm run dev
```

Клиент будет доступен на: **http://localhost:5173** (или другой порт, если 5173 занят)

---

## Альтернативный способ: Одновременный запуск (если есть корневой package.json)

Если в корне проекта есть `package.json` с настройкой `concurrently`:

```bash
# Установка зависимостей
npm run i          # или npm run install:all

# Запуск клиента и сервера одновременно
npm run dev
```

---

## Структура команд

### Сервер (server/package.json):
- `npm run dev` - Запуск в режиме разработки (nodemon + ts-node)
- `npm run build` - Компиляция TypeScript в JavaScript
- `npm start` - Запуск скомпилированного сервера
- `npm run start:dev` - Альтернативный запуск в режиме разработки

### Клиент (client/package.json):
- `npm run dev` - Запуск в режиме разработки (Vite)
- `npm run build` - Сборка production версии
- `npm run preview` - Предварительный просмотр production сборки
- `npm run lint` - Проверка кода линтером

---

## Порты по умолчанию

- **Клиент (Frontend)**: http://localhost:5173
- **Сервер (Backend)**: http://localhost:4000

## Важные замечания

1. **Сервер должен быть запущен до клиента**, так как клиент делает запросы к API сервера
2. **При первом запуске** убедитесь, что база данных настроена и доступна
3. **В режиме разработки** сервер перезагружается автоматически при изменении файлов благодаря nodemon
